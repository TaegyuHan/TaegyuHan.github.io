# 프로젝트 폴더 구조

<!-- file name : Untitled 3.png -->
<a href = "https://drive.google.com/uc?id=1jgFkbJ0h6g5Tjqpyg_be04iGDaFT96AL" target="_blank">
    <center><img src = "https://drive.google.com/uc?id=1jgFkbJ0h6g5Tjqpyg_be04iGDaFT96AL"></center>
    <center>[그림] 프로젝트 폴더 구조</center>
</a>




- `Practice/` : 프로젝트의 최상위 디렉토리 입니다. 이밖의 파일, 폴더는 프로젝트와는 연관이 없습니다.
- `Practice/manage.py` : 장고 프레임워크 명령어 실행 파일
- `Practice/Practice/` : 프로젝트를 위한 Python코드가 저장되어 있습니다.
- `Practice/Practice/__init__.py` : Folder를 모듈로 설정해주는 빈 Python 파일입니다.
- `Practice/Practice/asgi.py` : asgi는 ( Asynchronous Server Gateway Interface )의 줄임말 입니다. 즉 비동기 서버 인터페이스를 조작하는 파일입니다. 더 자세한 정보는 다른 포스트에서 다루도록 하겠습니다.
- `Practice/Practice/settings.py` : 현재 프로젝트의 환경설정을 담당합니다.
- `Practice/Practice/urls.py` : 현재 장고 프로젝트의 URL을 선언을 저장하고 다른 URL과 연결해줍니다.
- `Practice/Practice/wsgi.py` : wsgi는 ( Web Server Gateway Interface )약자로 웹서버와 웹 애플리케이션의 인터페이스를 위한 파이썬 프레임 워크 입니다.


브라우저에서 Django에게 바로 요청을 하는게 아니라 중간에 아파치, 엔진엑스의 서버가 존재하고 Django와 Sever를 연결시켜주는 WSGI의 역할을 알게되었습니다.

`Practice/Practice/asgi.py`

```python
"""
ASGI config for Practice project.

It exposes the ASGI callable as a module-level variable named ``application``.

For more information on this file, see
https://docs.djangoproject.com/en/4.0/howto/deployment/asgi/

import os # Python 내장 OS 모듈
from django.core.asgi import get_asgi_application

# 환경 변수 설정
os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'Practice.settings')

# asgi 어플리케이션 가져오기
application = get_asgi_application()
```

# 한눈에 정리하기

```python
└─Practice
    │  manage.py
    │  # 장고 프레임워크 명령어 실행 파일
    │ 
    └─Practice
            asgi.py # Asynchronous Server Gateway Interface 파일
            settings.py # 장고프레임워크 설정 파일
            urls.py # URL 경로 설정 파일
            wsgi.py # Web Server Gateway Interface 파일 
            __init__.py # Python 패키지, 모듈을 만들기위한 파일
```
